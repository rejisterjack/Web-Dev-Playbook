What's happening here?

Short Polling: The client repeatedly sends requests to the server at fixed intervals (every 3 seconds), regardless of whether new data is available. This results in many unnecessary requests but is simple to implement.

Long Polling: The client sends a request, and the server holds the connection open until new data is available or a timeout occurs. Once the client gets a response, it immediately sends another request. This reduces unnecessary requests while maintaining near real-time updates.
